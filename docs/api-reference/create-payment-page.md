# Create Payment Page - Référence Technique

## 📖 Description

L'endpoint `POST /v2/paymentpage` permet de créer une page de paiement hébergée par Pawapay. Cette page permet au client d'effectuer un paiement via Mobile Money sans avoir à intégrer directement l'API.

## 🔗 Endpoint

```
POST /v2/paymentpage
```

## 📋 Structure de la requête

### Headers

| Header | Valeur | Requis |
|--------|--------|--------|
| `Authorization` | `Bearer <token>` | ✅ Oui |
| `Content-Type` | `application/json` | ✅ Oui |
| `Accept` | `application/json` | ✅ Oui |

### Body (JSON)

| Champ | Type | Requis | Description |
|-------|------|--------|-------------|
| `depositId` | `string` | ✅ Oui | UUID v4 unique pour le dépôt |
| `returnUrl` | `string` | ✅ Oui | URL de redirection après le paiement |
| `amountDetails.amount` | `string` | ✅ Oui | Montant à payer |
| `amountDetails.currency` | `string` | ✅ Oui | Devise (ex: `USD`, `ZMW`) |
| `phoneNumber` | `string` | ✅ Oui | Numéro de téléphone du payeur (format E.164) |
| `language` | `string` | ✅ Oui | Langue de la page (`EN` ou `FR`) |
| `country` | `string` | ✅ Oui | Code pays ISO3 (ex: `COD`, `ZMB`) |
| `customerMessage` | `string` | ❌ Non | Message au client (4-22 caractères) |
| `reason` | `string` | ❌ Non | Raison du paiement |
| `metadata` | `array` | ❌ Non | Métadonnées personnalisées (max 10 champs) |

### Exemple de requête

```json
{
  "depositId": "f1e6cf9f-e492-4d15-b095-3ef49375a43f",
```

## 📊 Structure de la réponse

### Succès (200 OK)

```json
{
  "redirectUrl": "https://sandbox.paywith.pawapay.io/v2?token=xxx"
}
```

### Erreur - REJECTED (200 OK avec failureReason)

```json
{
  "failureReason": {
    "failureCode": "PROVIDER_TEMPORARILY_UNAVAILABLE",
    "failureMessage": "The provider 'VODACOM_MPESA_COD' is currently not able to process payments."
  }
}
```

```json
{
  "failureReason": {
```

## 💻 Utilisation avec le SDK

### Installation

```bash
composer require andydefer/php-pawapay
```

### Exemple avec Builder (recommandé)

```php
<?php

declare(strict_types=1);

require './vendor/autoload.php';

use AndyDefer\PhpPawapay\Builders\CreatePaymentPageBuilder;
use AndyDefer\PhpPawapay\Enums\Country;
use AndyDefer\PhpPawapay\Enums\Currency;
use AndyDefer\PhpPawapay\Enums\Language;
use AndyDefer\PhpPawapay\Enums\PawaPayBaseUrl;

// 1. Créer la page de paiement avec le Builder
$response = CreatePaymentPageBuilder::create(
    apiToken: 'your-api-token-here'
)
    ->withBaseUrl(PawaPayBaseUrl::SANDBOX)
    ->withAutoGeneratedDepositId()
    ->withPhoneNumber('243827833329')
    ->withAmount(24.60)
    ->withCurrency(Currency::USD)
    ->withReturnUrl('http://127.0.0.1:8000/payments/pawapay/success')
    ->withLanguage(Language::FR)
    ->withCountry(Country::COD)
    ->withCustomerMessage('paiement')
    ->withReason('paiement')
    ->withMetadataKey('orderId', 'ORD-123456789')
    ->withMetadataKey('customerId', 'customer@email.com')
    ->execute();

// 2. Traiter la réponse
if ($response->isSuccess()) {
    $redirectUrl = $response->getRedirectUrl();
    echo "✅ Page de paiement créée !\n";
    echo 'Redirect URL: ' . ($redirectUrl ? $redirectUrl->getValue() : 'N/A') . "\n";
} else {
    echo "❌ Échec de la création de la page de paiement\n";
    if ($response->hasFailureReason()) {
        $failure = $response->getFailureReason();
        echo "Code d'erreur: " . $failure->failureCode . "\n";
        echo 'Message: ' . $failure->failureMessage . "\n";
    }
}
```

### Exemple sans Builder (manuel)

```php
<?php

declare(strict_types=1);

require './vendor/autoload.php';

use AndyDefer\DomainStructures\Utils\StrictDataObject;
use AndyDefer\PhpClient\ValueObjects\UrlVO;
use AndyDefer\PhpPawapay\Enums\Country;
use AndyDefer\PhpPawapay\Enums\Currency;
use AndyDefer\PhpPawapay\Enums\Language;
use AndyDefer\PhpPawapay\Enums\PawaPayBaseUrl;
use AndyDefer\PhpPawapay\Graphs\AmountDetailsGraph;
use AndyDefer\PhpPawapay\PawapayClient;
use AndyDefer\PhpPawapay\Structures\PaymentPageStruct;
use AndyDefer\PhpPawapay\ValueObjects\AmountVO;
use AndyDefer\PhpPawapay\ValueObjects\MessageVO;
use AndyDefer\PhpPawapay\ValueObjects\MetadataVO;
use AndyDefer\PhpPawapay\ValueObjects\PhoneNumberVO;
use AndyDefer\PhpPawapay\ValueObjects\UuidVO;

// 1. Créer le client
$client = new PawapayClient(
    apiToken: 'your-api-token-here',
    baseUrl: PawaPayBaseUrl::SANDBOX
);

// 2. Créer les Value Objects
$depositId = new UuidVO('f1e6cf9f-e492-4d15-b095-3ef49375a43f');
$phoneNumber = new PhoneNumberVO('243827833329');
$amount = new AmountVO(24.60);
$returnUrl = new UrlVO('http://127.0.0.1:8000/payments/pawapay/success');

$amountDetails = new AmountDetailsGraph(
    amount: $amount,
    currency: Currency::USD
);

$metadata = new MetadataVO(
    new StrictDataObject([
        'orderId' => 'ORD-123456789',
        'customerId' => 'customer@email.com',
    ])
);

// 3. Créer la page de paiement
$paymentPage = new PaymentPageStruct(
    depositId: $depositId,
    returnUrl: $returnUrl,
    amountDetails: $amountDetails,
    phoneNumber: $phoneNumber,
    language: Language::FR,
    country: Country::COD,
    customerMessage: new MessageVO('paiement'),
    reason: new MessageVO('paiement'),
    metadata: $metadata
);

// 4. Créer la page de paiement
$response = $client->createPaymentPage($paymentPage);

// 5. Traiter la réponse
if ($response->isSuccess()) {
    $redirectUrl = $response->getRedirectUrl();
    echo "✅ Page de paiement créée !\n";
    echo 'Redirect URL: ' . ($redirectUrl ? $redirectUrl->getValue() : 'N/A') . "\n";
} else {
    echo "❌ Échec\n";
    if ($response->hasFailureReason()) {
        $failure = $response->getFailureReason();
        echo 'Code: ' . $failure->failureCode . "\n";
        echo 'Message: ' . $failure->failureMessage . "\n";
    }
}
```

## 🧩 Value Objects disponibles

| Value Object | Description | Validation |
|--------------|-------------|------------|
| `UuidVO` | UUID v4 | Format UUID v4 (génération auto si null) |
| `PhoneNumberVO` | Numéro de téléphone | Format E.164 |
| `AmountVO` | Montant | Positif, numérique |
| `MessageVO` | Message client | Normalisation + troncature à 22 caractères |
| `MetadataVO` | Métadonnées | Max 10 champs, valeurs scalaires, clés string |
| `UrlVO` | URL de retour | Validation format URL |

## 🌍 Enums disponibles

| Enum | Description |
|------|-------------|
| `Language` | Langue de la page (`EN`, `FR`) |
| `Country` | Code pays ISO3 (`COD`, `ZMB`, etc.) |
| `Currency` | Devises supportées (`USD`, `ZMW`, `KES`, etc.) |
| `PawaPayBaseUrl` | URL de base (`SANDBOX`, `PRODUCTION`) |
| `FailureCode` | Codes d'erreur de l'API |

## 🔧 Codes d'erreur

| Code | Description |
|------|-------------|
| `PROVIDER_TEMPORARILY_UNAVAILABLE` | Fournisseur indisponible |
| `INVALID_PHONE_NUMBER` | Numéro de téléphone invalide |
| `INVALID_CURRENCY` | Devise non supportée |
| `INVALID_AMOUNT` | Montant invalide |
| `AUTHENTICATION_ERROR` | Token API invalide |
| `AUTHORISATION_ERROR` | Token non autorisé |
| `UNKNOWN_ERROR` | Erreur inconnue |

## 🧪 Tests

```bash
./vendor/bin/phpunit --filter CreatePaymentPageTest
```

### Exemple de test

```php
public function test_create_payment_page_success(): void
{
    $this->client->addSuccessResponse([
        'redirectUrl' => 'https://sandbox.paywith.pawapay.io/v2?token=xxx',
    ]);

    $paymentPage = $this->createPaymentPageStruct();
    $response = $this->client->createPaymentPage($paymentPage);

    $this->assertTrue($response->isSuccess());
    $this->assertFalse($response->hasFailureReason());
    $this->assertNotNull($response->getRedirectUrl());
    $this->assertStringContainsString('sandbox.paywith.pawapay.io', $response->getRedirectUrl()->getValue());
}
```

## 📝 Notes importantes

1. **Redirection** : La réponse contient une URL de redirection vers la page de paiement Pawapay.

2. **Langue** : La page de paiement s'affiche dans la langue spécifiée (`EN` ou `FR`).

3. **Format du numéro de téléphone** : Utiliser le format E.164 sans le `+` (ex: `243827833329` pour la RDC).

4. **Validation des métadonnées** : Maximum 10 champs, valeurs scalaires uniquement.

5. **MessageVO** : Les messages sont automatiquement normalisés et tronqués à 22 caractères.

6. **Timeout** : Le SDK utilise un timeout de 30 secondes par défaut.
